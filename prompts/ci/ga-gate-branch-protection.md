# CI: GA Gate Composite Check Alignment

## Purpose

Align required checks policy and branch protection rules with the GA composite gate.

## Allowed Operations

- Create or edit CI governance artifacts for required checks and branch protection.
- Update roadmap status metadata for governance tracking.

## In Scope Paths

- docs/ci/REQUIRED_CHECKS_POLICY.yml
- docs/ci/REQUIRED_CHECKS_POLICY.json
- .github/protection-rules.yml
- docs/roadmap/STATUS.json
- agents/examples/
- prompts/registry.yaml

## Out of Scope

- Product code changes.
- Runtime or service configuration beyond branch protection policy.
