# Required Outputs for a Marketing System PR

## Files

- Updated artifacts in `marketing/`, `pitch/`, `sales/`, `press/`, or `governance/`.
- `governance/claims.registry.yaml` updated.
- `governance/claim-policy.md` updated if policy changes are needed.

## Reports (committed or included in the PR body)

- Consistency report (block/warn/info).
- Risk report (block/warn/info).
- Claim mapping (sentence -> claim ID).

## PR Body

Must include:

- Artifact index.
- Channel readiness.
- Gate checklist outcomes.
- CI outputs (or simulated results if CI is not available).
