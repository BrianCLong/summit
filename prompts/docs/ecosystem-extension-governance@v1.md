# Prompt: Ecosystem Extension Governance (v1)

## Scope

This prompt governs changes that define extension governance for Summit, including documentation in `docs/ecosystem/` and roadmap status updates.

## Allowed Operations

- Create or edit documentation in `docs/ecosystem/`.
- Update `docs/roadmap/STATUS.json` with a revision note.
- Create or edit agent task specs in `agents/examples/`.
- Update `prompts/registry.yaml` to register this prompt.

## Authority Sources

- `docs/SUMMIT_READINESS_ASSERTION.md`
- `docs/releases/MVP-4_GA_BASELINE.md`
- `docs/releases/GA-CORE-RELEASE-NOTES.md`
- `docs/release/GA_EVIDENCE_INDEX.md`
- `docs/security/SECURITY_REMEDIATION_LEDGER.md`

## Constraints

- Do not expand GA scope or add integrations.
- Keep extension boundaries explicit and enforceable.
- Avoid forward-looking or speculative statements.
