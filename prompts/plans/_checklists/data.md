# Data, ML & Experimentation Checklist

> Acceptance criteria for Prompts 36-40

## Prompt 36: Data Lineage and Provenance

- [ ] Lineage graph covers critical data flows
- [ ] Data contracts defined for key entities
- [ ] Visualization available and accessible
- [ ] Sensitive data flows highlighted
- [ ] Documentation auto-generated where possible
- [ ] Breaking change detection implemented
- [ ] Source-to-consumer mapping complete

## Prompt 37: Experimentation Platform Bootstrap

- [ ] A/B experiments can run in CI with results
- [ ] Experiment DSL validated
- [ ] Results statistically sound
- [ ] UI for viewing experiment results
- [ ] Documentation complete
- [ ] Routing/bucketing working correctly
- [ ] Guardrail metrics enforced

## Prompt 38: Mock Data Factory for Tests

- [ ] Factories work in all target languages (TS, Python)
- [ ] Deterministic output with seeds
- [ ] Shared fixtures validated against schema
- [ ] Tests run with deterministic data
- [ ] Documentation complete
- [ ] Cross-language compatibility verified
- [ ] Seed datasets created for all scenarios

## Prompt 39: Model/Experiment Reproducibility

- [ ] Reproduced experiment outputs match original
- [ ] Model registry operational
- [ ] Environment captured automatically
- [ ] Data versioning in place
- [ ] Reproducibility guide documented
- [ ] CI/CD integration for ML workflows
- [ ] Artifact storage configured

## Prompt 40: Data Ethics and Privacy Contingencies

- [ ] PII classified across all data stores
- [ ] Masking applied consistently
- [ ] GDPR/CCPA requests honored (tested)
- [ ] Audit trail complete
- [ ] Documentation compliant
- [ ] Privacy guardrails in code
- [ ] Deletion workflow functional
