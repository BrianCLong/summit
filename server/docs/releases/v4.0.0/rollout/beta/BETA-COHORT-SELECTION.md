## Overview

This document defines the selection criteria, target segments, and confirmed participants for the Summit v4.0 Beta program. The Beta phase expands testing from internal alpha testers to real customers in controlled production-like environments.

## Program Goals

1. **Validate v4 Features** - Test AI Governance, Cross-Domain Compliance, and Zero-Trust features with real customer workloads
2. **Scale Testing** - Verify platform performance under realistic multi-tenant usage
3. **Customer Feedback** - Gather actionable feedback to refine features before GA
4. **Migration Validation** - Test v3 â†’ v4 migration paths with real customer data
5. **Support Readiness** - Train support team through real customer interactions

## Selection Criteria

### Mandatory Requirements

| Criterion               | Requirement                                      | Rationale                             |
| ----------------------- | ------------------------------------------------ | ------------------------------------- |
| **Active Customer**     | Current Summit v3.x customer in good standing    | Known entities, existing relationship |
| **Technical Readiness** | Dedicated technical contact available            | Can provide timely feedback           |
| **Commitment**          | Signed Beta Agreement and NDA                    | Legal protection, clear expectations  |
| **Communication**       | Responsive to emails within 48 hours             | Effective collaboration               |
| **Environment**         | Non-production environment available for testing | Minimize risk to production workloads |

### Preferred Criteria (Weighted Scoring)

| Criterion                    | Weight | Description                                 |
| ---------------------------- | ------ | ------------------------------------------- |
| **Strategic Account**        | 3x     | Top-tier customers, reference potential     |
| **Feature Interest**         | 3x     | Expressed interest in v4 features           |
| **Compliance Need**          | 2x     | Active HIPAA, SOX, or SOC 2 requirements    |
| **AI Adoption**              | 2x     | Interested in AI-assisted governance        |
| **Geographic Diversity**     | 1x     | Coverage across regions                     |
| **Industry Diversity**       | 1x     | Coverage across verticals                   |
| **Technical Sophistication** | 2x     | Can provide detailed technical feedback     |
| **Migration Complexity**     | 2x     | Complex v3 setup tests migration thoroughly |

### Exclusion Criteria

- Customers in active support escalations
- Customers with outstanding billing issues
- Customers without a signed Beta Agreement
- Customers unable to commit resources for testing
- Customers requiring SLA guarantees during Beta
