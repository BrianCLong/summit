## Technical Onboarding Guide

### Pre-Requisites

| Requirement    | Details                                       |
| -------------- | --------------------------------------------- |
| **Browser**    | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| **Network**    | HTTPS access to \*.summit.io                  |
| **MFA**        | TOTP-compatible authenticator app             |
| **API Client** | Postman, cURL, or SDK (optional)              |

### Step 1: Environment Access

1. Navigate to **https://beta.summit.io**
2. Enter your credentials from the welcome email
3. Change your temporary password when prompted
4. Set up MFA using your authenticator app

### Step 2: Initial Configuration

After logging in, complete the setup wizard:

1. **Organization Profile**
   - Confirm organization name
   - Set timezone and locale
   - Upload logo (optional)

2. **Admin Users**
   - Invite additional beta testers
   - Assign appropriate roles

3. **Integration Settings**
   - Configure SSO if desired (optional for beta)
   - Set up API keys for integrations

### Step 3: Data Migration (Optional)

If testing migration from v3:

1. Navigate to **Settings → Migration**
2. Review pre-migrated data from your v3 tenant
3. Run validation checks
4. Note any migration issues for feedback

### Step 4: v4 Features Activation

Enable v4 features for testing:

| Feature         | Path                                    | Default  |
| --------------- | --------------------------------------- | -------- |
| AI Suggestions  | Settings → AI Governance → Suggestions  | Enabled  |
| AI Explanations | Settings → AI Governance → Explanations | Enabled  |
| HIPAA Module    | Settings → Compliance → HIPAA           | Disabled |
| SOX Module      | Settings → Compliance → SOX             | Disabled |
| Audit Ledger    | Settings → Security → Audit             | Enabled  |
| HSM Integration | Settings → Security → HSM               | Disabled |

### Step 5: API Access

```bash
# Set your API credentials
export SUMMIT_API_KEY="your-api-key"
export SUMMIT_API_URL="https://api-beta.summit.io/v4"

# Test API access
curl -H "Authorization: Bearer $SUMMIT_API_KEY" \
     "$SUMMIT_API_URL/health"

# Expected response:
# {"status":"healthy","version":"4.0.0-beta.X"}
```
