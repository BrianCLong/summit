# Rule: No Surprises

**Effective Date:** 2025-10-27
**Scope:** All Leadership Reporting & Escalations

## The Protocol

From now on, anything that could embarrass leadership or materially impact the strategic plan gets reported immediately using the following format.

**No drama. No speculation. Just control.**

### Reporting Format

1.  **What happened:** (Facts only, no emotion)
2.  **Impact:** (Quantifiable metric: user count, revenue, timeline delay, risk exposure)
3.  **Options:**
    *   Option A: ...
    *   Option B: ...
4.  **Recommendation:** (Which option and why)
5.  **Decision needed by when:** (Date/Time)

---

### Example

> **Subject:** [ESCALATION] Security Vulnerability in Agent Runtime
>
> **What happened:** A critical RCE vulnerability was discovered in the open-source dependency `xyz-lib` used by the Agent Runtime.
>
> **Impact:** Potential for full tenant compromise. Affects all Alpha users (3 active accounts).
>
> **Options:**
> *   Option A: Patch immediately, requires 2h downtime.
> *   Option B: Hotfix in place without downtime, but 24h engineering effort (delays feature X).
>
> **Recommendation:** Option A. Trust is our #1 bet. We cannot risk a breach during Alpha.
>
> **Decision needed by when:** Today, 14:00 EST.

---
**Enforcement:**
Reports that do not follow this format will be returned unread.
