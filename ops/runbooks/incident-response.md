# Incident Response Runbook

1. Page on-call and establish an incident commander.
2. Triage using SIEM and service dashboards; capture audit IDs and timelines.
3. Contain by revoking tokens/credentials, enabling WAF rules, and isolating affected workloads.
4. Eradicate by patching vulnerable components and rotating secrets.
5. Recover with controlled rollout, verification checks, and smoke tests.
6. Post-incident review within 72 hours including root cause, blast radius, and follow-up tasks.
