# Evidence Bundle

This directory contains deterministic evidence artifacts generated by
`summit_evidence.write_bundle`. Only `stamp.json` may include timestamps.

## Schemas

- `schemas/evidence/report.schema.json`: Schema for the main evidence report.
- `schemas/evidence/metrics.schema.json`: Schema for metrics collected during execution.
- `schemas/evidence/stamp.schema.json`: Schema for the attestation stamp (versions, time).
- `schemas/evidence/index.schema.json`: Schema for the index file mapping IDs to artifacts.

## Usage

Run `python ci/verify_evidence.py` to validate generated evidence against these schemas.

## Schema Validation

Validate these files against `schemas/evidence/*.schema.json`.

## Adding a new evidence bundle

1. Create a folder named `evidence/<EVIDENCE_ID>/` with `report.json`, `metrics.json`, and
   `stamp.json`.
2. Ensure only `stamp.json` contains timestamps.
3. Add the bundle paths to `evidence/index.json` under the `evidence` key.
   Example:
   ```json
   {
     "version": "1.0",
     "evidence": {
       "EVD-NEW-001": "path/to/evidence/folder"
     }
   }
   ```
4. Validate the bundle with the evidence verifier before committing changes.

## Evidence ID convention

Use the `EVD-SE-DICOW-<AREA>-<NNN>` format for SE-DiCoW-related runs.
