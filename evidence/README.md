# Evidence Bundle

This directory contains deterministic evidence artifacts generated by
Summit workflows. Only `stamp.json` may include timestamps.

## Schemas

- `schemas/evidence.report.schema.json`: Schema for the main evidence report.
- `schemas/evidence.metrics.schema.json`: Schema for metrics collected during execution.
- `schemas/evidence.stamp.schema.json`: Schema for the attestation stamp (versions, time).
- `schemas/evidence.index.schema.json`: Schema for the index file mapping IDs to artifacts.

## Usage

Run `tools/verify_evidence.py` to validate generated evidence against these schemas.

## Schema Validation

Validate these files against `schemas/*.schema.json`.

## Adding a new evidence bundle

1. Create a folder named `evidence/<EVIDENCE_ID>/` (or `<run_id>`) with `report.json`, `metrics.json`, and `stamp.json`.
2. Ensure only `stamp.json` contains timestamps.
3. Add the bundle paths to `evidence/index.json`.
4. Validate the bundle with the evidence verifier before committing changes.

### Evidence ID index entry format

Add a new entry under `items` in `evidence/index.json`.
Format: `EVD-<CONTEXT>-<AREA>-<NUMBER>` (e.g., `EVD-ENTR-2026-ECON-001`).

```json
{
  "version": "1.0",
  "items": {
    "EVD-EXAMPLE-001": {
        "files": [
          "evidence/EVD-EXAMPLE-001/report.json",
          "evidence/EVD-EXAMPLE-001/metrics.json",
          "evidence/EVD-EXAMPLE-001/stamp.json"
        ]
    }
  }
}
```

### Policies

**Stamp Isolation**: Timestamps must NOT appear in `report.json` or `metrics.json`.
