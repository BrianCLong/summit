# PR Policy

## Requirements for PR Creation

A PR must:

- Describe the motivation and context.
- List key changes (code, tests, docs).
- Indicate any breaking changes or migrations.
- Reference related issues or tasks if applicable.

## Review Standards

Reviewer must:

- Verify code correctness and edge cases.
- Ensure tests are adequate and green (conceptually, if not run).
- Confirm docs are updated.
- Check alignment with architecture and Prime Brain.

## Merge Criteria

A PR may be merged when:

- Reviewer verdict is **APPROVE**.
- No unresolved blocking comments remain.
- Changes adhere to governance and Prime Brain.
