# Post-Deploy Operations & Alert Rules

This directory contains Prometheus/Alertmanager alert rules and related operational checks focusing on the **first week** after a release. These alerts are designed to detect critical regressions, validation failures, and degraded health signals.

## First-Week Alerts (`alerts/first-week/first-week-alerts.yaml`)

### 1. CriticalErrorRateSpike
* **Severity:** Critical
* **Phase:** Post-Deploy
* **Description:** Triggers when the HTTP 5xx error rate exceeds 5% of all requests over a 5-minute rolling window, indicating a potential critical regression where the error rate is >5x baseline within an hour of deploy.
* **Operator Response:** Investigate recent deployments, application logs, and metrics. If a bad release is identified, trigger an immediate rollback.
* **Runbook:** [docs/runbooks/CriticalErrorRateSpike.md](../../docs/runbooks/CriticalErrorRateSpike.md)

### 2. FailedPostDeployValidation
* **Severity:** Critical
* **Phase:** Post-Deploy
* **Description:** Triggers when post-deployment validation checks (`scripts/post-deploy-validation/run-checks.sh`) fail. This implies critical smoke tests, system validations, or health endpoints are failing immediately post-release.
* **Operator Response:** Review the `post-deploy-results.json` artifact generated by the validation script. Identify failing components. Roll back the deployment if the issue cannot be resolved via configuration fix.
* **Runbook:** [docs/runbooks/FailedPostDeployValidation.md](../../docs/runbooks/FailedPostDeployValidation.md)

### 3. DegradedDependencyHealth
* **Severity:** Warning
* **Phase:** Post-Deploy
* **Description:** Triggers when downstream dependencies degrade (e.g., latency exceeds 2x SLA, or error rate surpasses 1%).
* **Operator Response:** Verify if the dependency outage is external or internal. Check for network partitions, timeouts, or changes in how the application queries dependencies. Escalate to the appropriate dependency owner if needed.
* **Runbook:** [docs/runbooks/DegradedDependencyHealth.md](../../docs/runbooks/DegradedDependencyHealth.md)

### 4. MissingOrInvalidEvidenceArtifact
* **Severity:** Critical
* **Phase:** Post-Deploy
* **Description:** Triggers when the required post-deploy evidence artifact is missing or invalid. Continuous compliance pipelines rely on this artifact to ensure software supply chain integrity.
* **Operator Response:** Ensure that CI/CD pipelines successfully generated and pushed the `evidence/index.json` or related artifacts. Rerun the artifact generation stage.
* **Runbook:** [docs/runbooks/MissingOrInvalidEvidenceArtifact.md](../../docs/runbooks/MissingOrInvalidEvidenceArtifact.md)

### 5. SLOBurnRateFastBurn
* **Severity:** Critical
* **Phase:** Post-Deploy
* **Description:** Triggers when the service is consuming its SLO error budget rapidly (e.g., burning through 5% of the total budget within 1 hour).
* **Operator Response:** Treat as an active incident. The release is severely degrading the user experience. Halt all rollouts, evaluate error logs, and consider a fast rollback or emergency mitigation strategy.
* **Runbook:** [docs/runbooks/SLOBurnRateFastBurn.md](../../docs/runbooks/SLOBurnRateFastBurn.md)

## Post-Deploy Validation Script

The script `scripts/post-deploy-validation/run-checks.sh` executes a suite of smoke tests and health checks.

* **Outputs:** Generates a `post-deploy-results.json` containing structured pass/fail results for each check.
* **Exit Codes:** Exits non-zero (`1`) if any critical check fails, allowing CI/CD pipelines to fail the job immediately.
