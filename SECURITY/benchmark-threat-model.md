# Benchmark Substrate Threat Model

## Interactive Benchmarks & Economies

### Threats & Mitigations

1. **Infinite Loops in Environments**
   - *Threat*: A poorly written agent gets stuck in a loop, consuming CI runners indefinitely.
   - *Mitigation*: Hard step budgets enforced by `BaseEnvironment.budget().stepsRemaining` and wallclock checks. The runner halts execution.

2. **Non-Determinism**
   - *Threat*: Evaluators returning different results, breaking regression testing and comparisons.
   - *Mitigation*: Use of fixed seeds (`config.seed`), strict `1970-01-01T00:00:00Z` timestamping inside trace logs, and pre-recorded golden fixture states.

3. **Prompt/Policy Drift**
   - *Threat*: Agentic guardrails bypass the benchmark sandbox.
   - *Mitigation*: Integration with existing `.opa/policy` rules. Sandboxed environment execution blocks arbitrary external access.

4. **Artifact Poisoning**
   - *Threat*: Corrupted or malicious telemetry generated by benchmark agents.
   - *Mitigation*: Strict JSON schema validation against `artifact/schemas/benchmark/interactive/*` for `report`, `metrics`, `stamp`, and `trace`.

5. **Economy Exploit Loops (Market Simulations)**
   - *Threat*: Agents discover arbitrage or exploit loopholes in simple market environments, breaking evaluations.
   - *Mitigation*: Deny-by-default trade execution policies (`transactionPolicy`), capped transactions, and invariant tracking.

6. **Unsafe Tool Use**
   - *Threat*: Benchmark agents executing live operations or unrestricted tool chains during evaluation.
   - *Mitigation*: Deny-by-default execution. Allowed tools must be strictly explicitly whitelisted.
