# ADR 0002: Benchmark Artifacts Contract

## Context
We need a consistent format and contract for benchmark evidence packs to validate our performance and correctness claims deterministically against our schemas and query shapes.

## Decision
The benchmark harness will output three primary artifacts for every run:
- `bench_report.json`: Details on the query execution, passes/failures, and workload summary.
- `bench_metrics.json`: Time-series and performance metrics (latency, throughput).
- `provenance.json`: Cryptographic evidence of the run context (dataset hash, configuration hash, Git commit SHA, timestamp, and environment metadata).

The harness must use a fixed random seed, a defined dataset hash, and explicitly versioned query shapes to guarantee that runs are reproducible.

## Consequences
- The benchmark artifacts must be generated by `make bench` and tracked in CI.
- Changes to performance behavior must be verifiable via diffs in the `bench_metrics.json` outputs.
