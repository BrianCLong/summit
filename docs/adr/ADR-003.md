# ADR-003: Retrieval Strategy: GraphRAG

## Status
Accepted

## Context
Standard Retrieval-Augmented Generation (pure RAG) relies on semantic similarity search over document chunks. While effective for simple fact retrieval, pure RAG often fails to capture structural relationships, multi-hop connections, and global context in complex intelligence datasets. Analysts need to understand not just *what* was said, but *how* entities are connected across disparate sources.

## Decision
We have decided to implement **GraphRAG** (Graph + Vector fusion) as the primary retrieval strategy for Summit.

This approach combines:
1. **Vector Retrieval**: Semantic search for relevant document chunks and entities.
2. **Graph Retrieval**: Multi-hop traversal to discover related entities and structural context that semantic search alone might miss.
3. **Fusion Layer**: A component that merges graph-based and vector-based evidence, deduplicates findings, and ranks them by combined relevance and saliency.

## Consequences
- **Positive**: Significantly improved reasoning over complex relationships, better handling of multi-hop queries, and higher-fidelity source attribution through graph-linked evidence.
- **Negative**: Increased system complexity; requires maintaining both a vector index and a high-fidelity knowledge graph. Higher latency for retrieval due to graph traversal and fusion steps.
- **Neutral**: Requires careful token budgeting to ensure that the combined graph and document context fits within LLM context windows.
