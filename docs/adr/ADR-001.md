# ADR-001: Choice of Graph Database

## Status
Accepted

## Context
Summit requires a robust, scalable, and high-performance graph database to manage complex intelligence data, including entities, relationships, and multi-hop trajectories. The system needs to support real-time querying, advanced graph algorithms (e.g., PageRank, Betweenness Centrality), and flexible schema evolution. Additionally, integration with AI/ML workflows and GraphRAG pipelines is essential.

## Decision
We have chosen **Neo4j 5.x** as the primary graph database for the Summit platform.

Neo4j was selected based on the following criteria:
1. **Performance**: High-performance relationship traversal for deep sub-graph expansion.
2. **Graph Data Science (GDS)**: Built-in support for advanced graph algorithms required for saliency scoring and influence detection.
3. **Query Language**: Cypher is a mature, expressive, and industry-standard query language for graph data.
4. **Ecosystem**: Strong integration with Node.js, Python, and existing AI/ML toolchains.
5. **Multi-tenancy**: Native support for multiple databases and tenant isolation within a single cluster.

## Consequences
- **Positive**: High query performance for multi-hop retrievals, access to advanced GDS algorithms, and a mature developer ecosystem.
- **Negative**: Adds operational complexity compared to simple relational databases; requires specialized knowledge in Cypher and graph modeling.
- **Neutral**: Requires dedicated infrastructure (Neo4j Aura or self-hosted clusters) and regular maintenance of graph projections for GDS.
