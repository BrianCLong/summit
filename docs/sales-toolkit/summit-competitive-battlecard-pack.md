## Summit Default Kill Shot

Use this in every qualified discovery and executive meeting:

> **Summit is not a tool. Summit is a governed investigation runtime.**
> It executes repeatable playbooks over an evidence-first intelligence graph and produces audit-ready, reproducible outputs.

### Five Disqualifying Questions

1. **Evidence + reproducibility**: Can every claim in a case file map to captured evidence and deterministic rerun steps?
2. **Playbooks-as-code**: Are investigative procedures versioned, approval-gated, and drift-monitored?
3. **Agentic execution**: Can guarded agents execute collect → normalize → fuse → analyze → report with provenance?
4. **Cross-domain fusion**: Can OSINT, internal enterprise data, and external risk signals share one entity model?
5. **Governance posture**: Can the platform prove policy enforcement for who ran what, why, and with which controls?

## Objection Handling (Executive + Procurement)

### "We already own tool X"

- Acknowledge sunk cost; do not force rip-and-replace as the first motion.
- Position Summit as the control and execution plane above existing tools.
- Offer a 30-day controlled pilot with one priority playbook and objective evidence gates.

### "Your competitor has broader source coverage"

- Reframe from source count to decision quality and reproducibility.
- Require claim-level provenance and deterministic replay in evaluation criteria.
- Demonstrate where Summit fuses external + internal context to raise confidence and reduce false positives.

### "Palantir can do everything"

- Confirm broad-platform capability while narrowing the mission to investigation runtime outcomes.
- Differentiate on deployment velocity, policy-gated playbooks, and audit-ready evidence exports.
- Offer coexistence pattern where Summit delivers fast mission capability while enterprise platform programs continue.

### "We need immediate risk reduction, not transformation"

- Use governed exception framing: start with a constrained lane (one mission, one playbook family, one approval policy).
- Commit to measurable cycle-time, reproducibility, and audit-prep improvements inside 90 days.
- Define explicit rollback criteria in the pilot SOW to de-risk adoption.

## Competitive Evaluation Protocol (Golden Path for Bakeoffs)

1. **Scenario lock:** Agree one mission scenario, fixed input dataset, and success metrics.
2. **Control lock:** Define policy roles, approval gates, and evidence bundle schema before demos.
3. **Execution run 1:** Baseline run with deterministic capture and operator notes.
4. **Execution run 2:** Replay same inputs and quantify output parity.
5. **Delta run:** Introduce controlled data changes and inspect rationale diffs.
6. **Governance review:** Validate policy logs, approvals, exceptions, and chain-of-custody artifacts.
7. **Decision memo:** Select platform based on evidence quality, reproducibility, and time-to-operationalization.

## UEF Evidence Bundle Checklist (for Pilot Exit)

- Run manifest: playbook version, policy version, runtime version, operator identity.
- Input manifest: data sources, ingest timestamps, connector hashes.
- Claim ledger: claim IDs mapped to source artifacts and extraction steps.
- Reproducibility report: rerun parity score and delta explanations.
- Governance report: approvals, denials, exceptions, and role attestations.
- Export package: signed archive and retention metadata.

## Packaging Summit Above Toolchains

### Default posture (recommended)

- Keep incumbent tools as data/visualization surfaces where useful.
- Position Summit as the governed investigation runtime across all feeds.
- Measure value by reproducibility, cycle time compression, and audit readiness.

### Replacement posture (when mandated)

- Triggered when buyer requests platform consolidation, policy centralization, or evidence standardization.
- Migrate high-volume playbooks first; retain edge connectors temporarily.
- Decommission point tools only after parity + control evidence verification.

## Competitive Proof Assets to Prioritize

1. **Playbook library:** 10–20 canonical investigation playbooks with versioning and approvals.
2. **Evidence bundle export:** cryptographic hashes, capture metadata, citations, and replay instructions.
3. **Replay + diff:** deterministic reruns and change rationale outputs.
4. **Policy gates:** role, environment, and execution-policy controls with immutable logs.

## MAESTRO Security Alignment (for customer architecture reviews)

- **MAESTRO layers impacted:** Data, Agents, Tools, Observability, Security.
- **Threats considered:** prompt injection, source poisoning, tool abuse, policy bypass, evidence tampering.
- **Mitigations:** policy-gated execution, immutable provenance ledger, deterministic replay checks, least-privilege tool contracts, anomaly observability hooks.
