# Bellingcat OSINT Toolkit Normalization Standard

This standard defines the schema and guidelines for ingesting the public Bellingcat OSINT toolkit into the Summit platform.

## Purpose

The Bellingcat OSINT toolkit provides a comprehensive listing of 100+ tools used in open-source investigations. Integrating this toolkit into Summit ensures that analysts have access to a standardized, up-to-date registry of tools.

## Schema Definition

Tools ingested from Bellingcat (and other sources) MUST be normalized into the following schema format:

```json
{
  "id": "deterministic-hash-or-slug",
  "name": "Tool Name",
  "category": "Tool Category",
  "url": "https://example.com/tool",
  "description": "Short description of what the tool does and its limitations.",
  "source": "bellingcat",
  "authNeeded": false,
  "cost": "free",
  "lastChecked": "YYYY-MM-DD"
}
```

## Import Guidelines

- **Determinism**: Every imported tool must have a strictly deterministic `id` generated by `normalizeToolId(source, name)`. Output bundles (`artifacts/toolkit/bellingcat.json`) must be deterministically sorted by `id`.
- **Category Normalization**: Source-specific categories should be mapped to Summit's standard OSINT categories if applicable, but preserving the original category is acceptable in the minimum winning slice (MWS).
- **Update Frequency**: The toolkit should be periodically synced (via the `summit toolkit sync` command) to reflect additions or removals.
- **Drift Detection**: Any tools imported must eventually support health checking (e.g., verifying URLs return 200 OK) to ensure the registry remains reliable. This is managed by the daily drift detector.

## Security & Policy

- **Deny-by-Default**: Networked tools are off by default in workflows. Importing a tool only registers its metadata; it does not grant the platform permission to actively query it unless explicitly enabled via feature flags.
- **No Private Data Harvesting**: The tools registry is strictly public OSINT. It must not be used to construct workflows that violate ToS (e.g., scraping authenticated platforms).
