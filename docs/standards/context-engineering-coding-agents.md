# Standard: Context Engineering for Coding Agents

**Status:** Draft
**Version:** 1.0.0
**Owner:** Jules (Release Captain)

## 1. Overview

This standard defines how Summit constructs, validates, and reports the context (prompts, rules, interfaces) provided to AI coding agents. It transforms "prompt engineering" from an ad-hoc activity into a **governed, deterministic software artifact**.

### 1.1 Goals

1.  **Determinism**: Every agent run produces a `context_report.json` stating exactly what rules were loaded.
2.  **Budgeting**: Context size is explicitly budgeted; runs exceeding limits fail fast.
3.  **Provenance**: Every rule trace back to a specific file and version.
4.  **Security**: "Never-log" patterns are enforced at the context assembly layer.

## 2. Taxonomy

We divide context into four distinct layers, stored in `.summit/context/`:

| Layer | Directory | Purpose | Mutability |
| :--- | :--- | :--- | :--- |
| **Guidance** | `guidance/` | "Always-on" rules. Project conventions, style guides, critical security invariants. | **Low** (Governed) |
| **Instructions** | `instructions/` | Task-specific prompts. "How to run the linter", "How to add a migration". Loaded on demand. | **High** |
| **Interfaces** | `interfaces/` | Read-only schemas or API signatures exposed to the agent. | **Medium** |
| **Skills** | `skills/` | Executable tool definitions (if enabled). | **Low** (Strictly Allowlisted) |

## 3. Artifacts

### 3.1 `context_manifest.json`
A static inventory of all available context files in the repo.
Generated by: `pnpm summit-context build`

### 3.2 `context_report.json`
A runtime record of what was *actually* loaded for a specific session.
Must include:
- `sessionId`: Unique run ID.
- `loaded`: List of file IDs.
- `budget`: Token/char usage vs limit.
- `provenance`: Git SHA of the context files.

## 4. Governance & Gates

### 4.1 Budget Gate
CI will fail if the default "Guidance" bundle exceeds **40k characters**. This forces conciseness.

### 4.2 Policy Gate
Context files are scanned for:
- **Secrets**: API keys, tokens (deny).
- **Ambiguity**: "Do your best" (warn).
- **Conflicts**: Direct contradictions between files (warn).

## 5. Integration

This standard integrates with `AGENTS.md` by serving as the implementation detail for the "Context Engineering" section.
