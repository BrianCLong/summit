# Summit Agent Evaluation Protocol

## Overview
Summit Bench defines a deterministic execution protocol and evaluation standard for AI agents. It serves as the "MLPerf for AI agents", ensuring reproducible results, verifiable tool usage, and governance-scored benchmarks.

## Architecture
1. **Summit Protocol**: A standard set of JSON artifacts emitted by any conforming agent run.
2. **Deterministic Replay**: A verification engine that guarantees the reproducibility of execution traces.
3. **Benchmarks**: Standardized testing suites (e.g., GraphRAG) that evaluate reasoning, grounding, and policy compliance.
4. **Adapters**: Lightweight interfaces that map external frameworks (LangGraph, AutoGen) to the Summit protocol.

## Required Artifacts
An agent system must emit the following deterministic JSON artifacts complying with Draft 2020-12 schemas (located in `adk/spec/`):

1. `trace.json`: The step-by-step execution path and tool invocations.
2. `plan.json`: The task DAG generated by the agent planner.
3. `metrics.json`: Quantitative outcomes (e.g., token usage, step count, precision).
4. `report.json`: The final result, narrative summary, and overall governance score.
5. `stamp.json`: Environmental metadata (hashes, nonces) for deterministic replay.

**Important Data Handling Rule:** With the exception of `stamp.json`, these artifacts must **not** contain wall-clock timestamps or random UUIDs that break deterministic reproducibility.

## Evidence IDs
Summit utilizes strict Evidence IDs to map findings and context across outputs.
Format: `EVID:<source-type>:<source-id>:<content-hash>`
Example: `EVID:dataset:graphrag_001:abcd123`
