# Post-GA Change Control Policy

## Overview

After GA, stability is paramount. All changes must adhere to this policy.

## Change Classification

| Class     | Description                         | Requirements                                             |
| --------- | ----------------------------------- | -------------------------------------------------------- |
| **Patch** | Bug fixes, non-breaking chores.     | 1 Reviewer, CI Pass.                                     |
| **Minor** | New features (backward compatible). | 2 Reviewers, E2E Tests, Feature Flag.                    |
| **Major** | Breaking changes.                   | Architecture Review, Migration Plan, Major Version Bump. |

## Agent-Generated Changes

- All code generated by AI agents must undergo **human review**.
- Agents must include a "Reasoning Trace" in the PR description.
- Automated commits to `main` are **forbidden**.

## Velocity Guardrails

- **Code Freeze**: No Minor/Major changes on Fridays.
- **Canary**: Minor/Major changes must bake in Canary for 24h.
