## Quick Start by Role

### 👨‍💻 For Developers

**1-Minute Setup**:
```bash
# Install everything
npm run s-aos:install

# Check status
npm run s-aos:status

# Try a commit (template auto-loads)
git commit
```

**Daily Workflows**:
```bash
# Check last commit
npm run s-aos:check:commit

# Generate PR checklist
npm run s-aos:pr-checklist > PR.md

# View examples
npm run s-aos:examples
```

**VS Code Shortcuts**:
- Type `s-aos-commit` → Full commit template
- Type `s-aos-pr` → Full PR description
- Type `s-aos-feat` → Quick feature commit
- Type `s-aos-fix` → Quick bug fix commit

### 🏗️ For Platform Teams

**1-Day Deployment**:
```bash
# 1. Run deployment script
./scripts/deploy-s-aos-improvements.sh --environment=staging

# 2. Install dependencies
pnpm install

# 3. Test everything
npm run s-aos:test
npm run s-aos:benchmark

# 4. Verify compliance
npm run s-aos:check
```

**Weekly Operations**:
```bash
# Generate weekly report
npm run s-aos:report:weekly

# Check audit health
npm run s-aos:check:audit

# Update badges
npm run s-aos:badge --type=compliance --score=$(calculate_score)
```

### 👔 For Management

**Monthly Reviews**:
```bash
# Generate monthly report
npm run s-aos:report:monthly

# Output: reports/s-aos/s-aos-report-YYYY-MM-DD.md
```

**Report Contents**:

## GitHub Actions Reusable Workflow

### Using in Other Repos

```yaml
# .github/workflows/s-aos.yml
name: S-AOS Compliance

on:
  pull_request:
  push:
    branches: [main]

jobs:
  compliance:
    uses: your-org/summit/.github/workflows/s-aos-reusable-compliance.yml@main
    with:
      node-version: '20'
      skip-audit-check: false
    secrets:
      audit-secret: ${{ secrets.AUDIT_LOG_SECRET }}
```

### Workflow Outputs

```yaml
- name: Check compliance result
  run: |
    echo "Score: ${{ needs.compliance.outputs.compliance-score }}"
    echo "Passed: ${{ needs.compliance.outputs.passed }}"
```
