# Summit Architecture Constitution

## Architectural Rules for Evolution

To protect Summit from future architectural drift and ensure long-term stability toward GA, the following rules are mandated for all future development:

1.  **Evidence Mandate**: All entities and intelligence discoveries must reference verifiable evidence artifacts. No claim exists without a trace.
2.  **Canonical Identity**: All graph edges and relationship references must use canonical IDs only. This prevents entity explosion and ensures graph scalability.
3.  **Validation Gate**: Autonomous agents cannot modify the IntelGraph without explicit validation and verification steps (automated or human-in-the-loop).
4.  **Control Loop Integrity**: Core RepoOS control loops (homeostasis, health monitoring) cannot be disabled or bypassed in production environments.
5.  **Determinism**: All intelligence artifacts (reports, briefs, alerts) must be deterministic and reproducible given the same input signals and belief state.
6.  **Explainability**: Every decision and forecast must include a reasoning trace, decision path, and evidence chain.

These rules are non-negotiable and form the stability substrate for Summit's evolution.
