# Governance Exceptions

**Status:** Active | **Last Reviewed:** 2025-05-12

This document outlines the process for requesting and managing exceptions to governance policies.

## Exception Process

1.  **Request:** Open a specialized Issue using the "Governance Exception" template.
2.  **Review:** The Governance Board reviews the risk and justification.
3.  **Approval:** Requires sign-off from Security and the relevant Domain Owner.
4.  **Implementation:**
    *   Permanent exceptions are codified in `docs/governance/policies/exceptions.yml`.
    *   Temporary waivers are tracked in the `waivers/` directory with an expiration date.
