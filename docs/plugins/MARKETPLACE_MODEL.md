# Marketplace Model

## Submission Process

1.  **Develop**: Create and test your plugin.
2.  **Submit**: Upload to the Marketplace portal.
3.  **Review**:
    - **Automated Scan**: Security checks, linting.
    - **Manual Review**: Functional verification (for public plugins).
4.  **Publish**: Available in the catalog.

## Trust Tiers

- **Internal**: Developed by the tenant's own team. Implicit trust (but still sandboxed).
- **Verified**: Developed by Summit or certified partners. Audited for security.
- **Community**: Open submissions. Use at your own risk.

## Ratings & Feedback

(Future) Tenants can rate plugins and report issues.

## Monetization

(Future) Support for paid plugins via subscription or usage-based billing.
