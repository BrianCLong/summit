# Runbook - 2602.16742 Reproduction and Drift Monitoring

## Scope

Operational steps for running the `2602.16742` evaluator gate and weekly drift monitor.

## Reproduce Claim Locally

1. Create observed metric input:

```bash
cat > /tmp/2602-metrics.json <<'JSON'
{"qwen3_vl_8b_deepvision_wemath_pass_at_1": 85.11}
JSON
```

2. Run evaluator:

```bash
PYTHONPATH=. python3 -m summit.evals.paper_2602_16742 \
  --metrics-file /tmp/2602-metrics.json \
  --out-dir artifacts/2602.16742/local \
  --evidence-id EVD-2602-16742-EVAL-LOCAL-001 \
  --target-value 85.11 \
  --tolerance 0.5 \
  --fail-on-miss
```

3. Inspect generated artifacts:

- `artifacts/2602.16742/local/report.json`
- `artifacts/2602.16742/local/metrics.json`
- `artifacts/2602.16742/local/stamp.json`
- `artifacts/2602.16742/local/evidence.json`

## Weekly Drift Check

1. Prepare baseline and current metrics JSON files.
2. Run monitor:

```bash
python3 scripts/monitoring/paper_2602_16742_drift.py \
  --baseline artifacts/2602.16742/baseline.json \
  --current artifacts/2602.16742/current/metrics.json \
  --out-dir artifacts/2602.16742/weekly \
  --metric-key qwen3_vl_8b_deepvision_wemath_pass_at_1 \
  --threshold 0.5
```

3. Review outputs:

- `artifacts/2602.16742/weekly/trend.json`
- `artifacts/2602.16742/weekly/variance.json`
- `artifacts/2602.16742/weekly/alert.log`

## Failure Triage

1. If evaluator exits non-zero:
   - Check metric key name and input payload shape.
   - Confirm observed value source and tolerance configuration.
2. If drift monitor fails:
   - Compare `baseline_value` and `current_value`.
   - Verify benchmark runner/config parity and seed consistency.
   - Open an issue with artifact links and metric delta.

## Alert Thresholds

- Drift alert if absolute deviation exceeds `0.5` on the tracked metric.
- CI gate fail on `--fail-on-miss` when reproduction is outside tolerance.
