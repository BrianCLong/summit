## Issue 3: Epistemic Feedback Loop: Capture Analyst Overrides

**Title:** Epistemic Feedback Loop: Capture Analyst Overrides

**Description:**
Implement the feedback loop that allows analysts to override the Maestro agent's initial assessment. This is critical for training the agent over time.

**Acceptance Criteria:**
- [ ] Wire up the "Approve", "Downgrade", and "Block" buttons in the Right Pane to backend API endpoints.
- [ ] Create a new endpoint `POST /api/v1/leads/:id/feedback` to handle these override actions.
- [ ] **IntelGraph Integration:** When an override occurs, create a new *epistemic event* claim in the IntelGraph associating the analyst's decision (and optional justification) with the original lead claim.
- [ ] Update the lead's status in the UI to reflect the analyst's decision immediately.
- [ ] **Metric Target:** Ensure the override event is successfully recorded in the database 100% of the time.
