# Deterministic vs Runtime Stamps

## Overview

Summit CI separates deterministic build fingerprints from run-specific metadata. Deterministic
stamps are stable across reruns of the same commit, while runtime stamps capture ephemeral run
metadata.

## Deterministic Stamp

Location: `artifacts/deterministic/stamp.json`

Properties:

- No timestamps
- Stable key ordering
- Contains commit SHA, Dockerfile hash, lockfile hash, and toolchain versions

Schema: `schemas/evidence/deterministic_stamp_v1.jsonschema`

## Runtime Stamp

Location: `artifacts/runtime/runtime-stamp.json`

Properties:

- Includes run IDs, runner metadata, and timestamps
- Never used for deterministic comparisons

Generated by `tools/ci/emit_stamps.sh`.
