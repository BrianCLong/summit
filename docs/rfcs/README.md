# RFCs (Requests for Comments)

This directory contains proposals for significant changes to our policies, workflows, and architecture.

## RFC Lifecycle
1. **Draft**: The RFC is being written and is not yet ready for formal review.
2. **Review**: The RFC is open for discussion and feedback from the community.
3. **Approved**: The RFC has been accepted and is ready for implementation.
4. **Implemented**: The changes proposed in the RFC have been fully integrated.
5. **Superseded/Retired**: The RFC is no longer active or has been replaced.
