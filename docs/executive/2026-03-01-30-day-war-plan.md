## 6. Risk Mitigation Brief

**Catastrophic Risks:**
1. **Tenant Data Leak:** If tenant A sees tenant B's data, we lose all credibility.
   - *Mitigation:* Implement strict RLS (Row-Level Security) in Postgres, separate Neo4j databases per tenant, and automated daily penetration tests against the API layer.
2. **Agentic "Runaway" Actions:** An agent executing a tool in a loop or performing destructive actions.
   - *Mitigation:* Hard-coded rate limits, cost budgets per session, and mandatory human-in-the-loop (HITL) approval for any "write" action.
3. **Loss of Provenance:** The audit log is corrupted or deleted.
   - *Mitigation:* Append-only ledger architecture, off-site encrypted backups of the ledger, and cryptographic hashing of all evidence chains.

**Fundability Blockers:**
1. **Lack of predictable revenue.**
   - *Mitigation:* Force the High-ticket AI Ops offer. Secure at least 2 paid pilots in 30 days.
2. **Inability to demonstrate margin/COGS.**
   - *Mitigation:* Complete the FinOps attribution dashboard (Priority 3). Show investors exactly what it costs to service one tenant.
3. **Unclear competitive differentiation.**
   - *Mitigation:* Aggressively market the "Governed MCP" and "Deterministic Evidence" features. Make "Security as a Feature" our wedge.

**Hidden Margin Killers:**
1. **Unbounded LLM API calls.**
   - *Mitigation:* Implement strict token budgets per tenant and per agent run. Cache common GraphRAG queries.
2. **Orphaned infrastructure.**
   - *Mitigation:* Implement automated sweeping of unused development environments and orphaned cloud resources.
3. **High customer onboarding time.**
   - *Mitigation:* Standardize the 3 "golden path" templates so a new tenant can be provisioned and functional in < 1 hour.
