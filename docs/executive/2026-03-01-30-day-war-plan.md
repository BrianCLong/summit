# 30-Day War Plan: Project Eclipse

**Strategic Objective:** Execute the "Eclipse" strategy to aggressively position Summit CompanyOS against legacy threat intelligence solutions (SpiderFoot, Recorded Future) by establishing minimal viable parity and launching our core differentiator: the Proof Moat.

**Target ICP:** Enterprise Threat Intelligence & Investigations (Legal/Insider Risk)

## Week 1: Parity Kernel Foundation & Moat Scaffolding

**Objective:** Lay the groundwork for the Module SDK and begin designing the Evidence Integrity Gate.

*   **Codex (Parity):**
    *   Initialize the Module Ecosystem SDK utilizing our MCP-first architecture.
    *   Draft initial correlation rules engine specifications.
*   **Antigravity (Moat):**
    *   Complete the technical design specification for the EVID tag system and deterministic artifact ID generation.
    *   Define the provenance capture and lineage tracking database schema.
*   **Palette (UX):**
    *   Begin wireframing the "Case-First" investigation UX, focusing on the evidence capture interface and run-to-run diff visualizations.

## Week 2: Continuous Drift & Integration Stubs

**Objective:** Implement scheduling for drift detection and start building integration bridges to standard enterprise tooling.

*   **Codex (Parity):**
    *   Develop the scheduling engine for continuous drift detection runs.
    *   Implement initial data export utilities (JSON, CSV).
    *   Build bi-directional integration stubs for Jira and ServiceNow (Ticketing).
*   **Antigravity (Moat):**
    *   Implement core logic for hash-stable output generation, ensuring run-to-run reproducibility.
*   **Palette (UX):**
    *   Finalize wireframes and begin frontend implementation of the collaborative hypothesis workspaces.

## Week 3: Core Modules & Cryptographic Signatures

**Objective:** Launch the first two table-stakes modules and implement the verifiable signing process.

*   **Codex (Parity):**
    *   Implement the core Vulnerability Intelligence module (prioritizing exploitability/exposure).
    *   Implement the core Brand Protection module (impersonation, typosquatting).
    *   Build bi-directional integration stubs for major SIEM/SOAR platforms (Splunk, Cortex).
*   **Antigravity (Moat):**
    *   Finalize and integrate the verifiable signing process for all generated evidence bundles.
*   **Palette (UX):**
    *   Implement the executive report generation tool, automatically linking claims to EVID citations.

## Week 4: Integration, Hardening, & Enablement

**Objective:** Integrate the Parity Kernel, Proof Moat, and UX layers. Conduct rigorous testing and prepare GTM materials.

*   **Codex/Antigravity/Palette:**
    *   End-to-end integration testing: Run full intelligence pipelines from collection through to signed executive reports.
    *   Bug fixing and performance optimization.
*   **Jules (Enablement):**
    *   Draft updated battlecards and competitive matrices highlighting the completed Eclipse capabilities against SpiderFoot and Recorded Future.
    *   Prepare internal enablement materials for sales and solutions engineering on pitching the "Actionability + Provability" differentiator.
