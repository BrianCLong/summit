# Contributing Documentation

Welcome to the detailed contributing documentation for the Summit platform!

This directory contains in-depth guides to help you set up your local development environment, understand our code standards, and successfully contribute features and fixes to Summit.

## Table of Contents

1. [Development Setup & Repository Structure](setup.md)
   - Learn about our monorepo structure.
   - Set up your Node.js and Python environments.
   - Configure your local database using Docker Compose.

2. [Testing Locally](testing.md)
   - Run unit, E2E, and integration tests across the stack.
   - Understand the evaluation harnesses for GraphRAG.
   - Ensure the "Golden Path" remains stable.

3. [Code Style & Contribution Standards](standards.md)
   - Follow our strict conventional commits and branch naming policies.
   - Learn the PR process, including evidence bundles and CI gates.
   - Review guidelines for external agents and documentation.

4. [Contribution Workflow & Architecture Guidelines](workflow.md)
   - Add new graph entity types safely.
   - Implement new query patterns and their mock counterparts.
   - Debug common issues related to dependencies, testing, and CI pipelines.

For the high-level contribution philosophy and the quick-start guide, please refer to the main [`CONTRIBUTING.md`](../../CONTRIBUTING.md) at the root of the repository.
