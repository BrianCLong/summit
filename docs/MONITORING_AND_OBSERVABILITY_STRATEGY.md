# Summit Monitoring and Observability Strategy

**Version**: 1.0  
**Date**: January 30, 2026  
**Status**: Active  
**Audience**: Engineering, DevOps, SRE, Operations

---

## Executive Summary

This strategy establishes Summit's comprehensive monitoring and observability framework, enabling rapid detection of issues, deep root cause analysis, and continuous optimization. The framework covers metrics collection, logging aggregation, distributed tracing, alerting, and dashboarding across all services.

---

## 1. Core Observability Pillars

### 1.1 Metrics (Time-Series Data)

**Definition**: Quantitative measurements of system behavior over time

**Collection Points**:
- Application metrics (request rate, latency, error rate)
- - Infrastructure metrics (CPU, memory, disk, network)
  - - Database metrics (connection pool, query latency, replication lag)
    - - Business metrics (users online, transactions, revenue)
     
      - **Tools**: Prometheus, Datadog, custom instrumentation
      - **Retention**: 15 days (high resolution), 1 year (aggregated)
      - **Collection Interval**: 30 seconds (standard), 10 seconds (critical paths)
     
      - ### 1.2 Logs (Event Data)
     
      - **Definition**: Timestamped records of application and system events
     
      - **Log Levels**:
      - - ERROR: Unexpected failures requiring attention
        - - WARN: Potentially problematic situations
          - - INFO: Notable business/operational events
            - - DEBUG: Diagnostic information (development only)
             
              - **Tools**: ELK Stack (Elasticsearch, Logstash, Kibana), Datadog
              - **Retention**: 30 days (hot), 90 days (cold), 1 year (archive)
              - **Volume Target**: < 10GB/day across all services
             
              - ### 1.3 Traces (Request Flow)
             
              - **Definition**: Detailed record of request execution across services
             
              - **Capture Points**:
              - - API gateway entry point
                - - Each microservice boundary
                  - - Database queries
                    - - External API calls
                      - - Cache operations
                       
                        - **Tools**: Jaeger, Datadog APM
                        - **Retention**: 7 days
                        - **Sampling**: 10% standard, 100% for errors/slow requests
                       
                        - ### 1.4 Profiles (Resource Usage)
                       
                        - **Definition**: CPU, memory, and I/O usage patterns
                       
                        - **Collection Points**:
                        - - CPU profiling (sample every 10ms)
                          - - Memory profiling (allocations and heap)
                            - - I/O profiling (disk/network operations)
                             
                              - **Tools**: Pprof (Go), Pyflame (Python), Node.js profiler
                              - **Retention**: 7 days
                              - **Frequency**: Continuous (low overhead)

                              ## 2. Metrics Strategy

                              ### 2.1 Key Metrics by Category

                              **Availability**:
                              - Uptime percentage (target: 99.95%)
                              - - Error rate by severity (target: < 0.1% for P1/P2)
                                - - Health check status (binary)
                                 
                                  - **Performance**:
                                  - - Request latency: p50, p95, p99 (target: p99 < 300ms)
                                    - - Throughput: requests/sec (baseline + 10%)
                                      - - Database query latency: p99 (target: < 100ms)
                                       
                                        - **Saturation**:
                                        - - CPU utilization (target: < 70% peak)
                                          - - Memory usage (target: < 80% peak)
                                            - - Disk usage (target: < 85% capacity)
                                              - - Database connection pool (target: < 75% used)
                                               
                                                - **Errors**:
                                                - - Error rate by type (4xx, 5xx, timeout)
                                                  - - Exception rate by service
                                                    - - Failed deployments
                                                      - - Rollback frequency
                                                       
                                                        - **Business**:
                                                        - - Active users
                                                          - - Transaction count
                                                            - - Revenue processed
                                                              - - Feature usage by segment
                                                               
                                                                - ### 2.2 Metric Instrumentation
                                                               
                                                                - **Application Level**:
                                                                - ```
                                                                  # Request metrics
                                                                  request_duration_seconds{method, endpoint, status}
                                                                  request_total{method, endpoint, status}
                                                                  request_in_progress{method, endpoint}

                                                                  # Business metrics
                                                                  transactions_total{status, type}
                                                                  users_active{region, plan}
                                                                  revenue_total{currency}

                                                                  # Error tracking
                                                                  errors_total{type, service, severity}
                                                                  ```

                                                                  **Infrastructure Level**:
                                                                  ```
                                                                  # Host metrics
                                                                  node_cpu_seconds{mode, cpu}
                                                                  node_memory_bytes{state}
                                                                  node_network_bytes{device, direction}

                                                                  # Container metrics (if applicable)
                                                                  container_cpu_usage_seconds
                                                                  container_memory_usage_bytes
                                                                  container_network_io_bytes
                                                                  ```

                                                                                                      ## 3. Logging Strategy
                                                                                                      
                                                                                                      ### 3.1 Log Structure
                                                                                                      
                                                                                                      **Standard Format** (JSON):
                                                                                                      ```json
                                                                                                      {
                                                                                                        "timestamp": "2026-01-30T16:32:45.123Z",
                                                                                                        "level": "ERROR",
                                                                                                        "service": "api-service",
                                                                                                        "host": "api-prod-1",
                                                                                                        "request_id": "req-12345",
                                                                                                        "user_id": "user-567",
                                                                                                        "message": "Payment processing failed",
                                                                                                        "error": {
                                                                                                          "type": "PaymentGatewayError",
                                                                                                          "message": "Connection timeout",
                                                                                                          "stack_trace": "..."
                                                                                                        },
                                                                                                        "context": {
                                                                                                          "order_id": "order-789",
                                                                                                          "amount": 99.99,
                                                                                                          "retry_count": 2
                                                                                                        }
                                                                                                      }
                                                                                                      ```
                                                                                                      
                                                                                                      ### 3.2 Log Levels and Guidelines
                                                                                                      
                                                                                                      **ERROR**:
                                                                                                      - Application exceptions (catch clause)
                                                                                                      - - External service failures (after retries)
                                                                                                        - - Data validation failures
                                                                                                          - - All security-related issues
                                                                                                            - - **Volume target**: < 1% of all logs
                                                                                                             
                                                                                                              - **WARN**:
                                                                                                              - - Degraded performance (> 2x baseline latency)
                                                                                                                - - High resource usage (> 80% capacity)
                                                                                                                  - - Deprecated API usage
                                                                                                                    - - Unusual but recoverable conditions
                                                                                                                      - - **Volume target**: 5-10% of all logs
                                                                                                                       
                                                                                                                        - **INFO**:
                                                                                                                        - - Significant business events (user signup, payment)
                                                                                                                          - - Service startup/shutdown
                                                                                                                            - - Configuration changes
                                                                                                                              - - Deployment events
                                                                                                                                - - **Volume target**: 30-40% of all logs
                                                                                                                                 
                                                                                                                                  - **DEBUG**:
                                                                                                                                  - - Method entry/exit (development only)
                                                                                                                                    - - Variable state dumps
                                                                                                                                      - - Detailed execution flow
                                                                                                                                        - - Third-party library logs
                                                                                                                                          - - **Volume target**: 0% in production
                                                                                                                                           
                                                                                                                                            - ### 3.3 Structured Logging
                                                                                                                                           
                                                                                                                                            - **Required Fields** (all logs):
                                                                                                                                            - - timestamp (ISO 8601)
                                                                                                                                              - - level (ERROR, WARN, INFO, DEBUG)
                                                                                                                                                - - service (service name)
                                                                                                                                                  - - request_id (trace ID)
                                                                                                                                                    - - message (human-readable)
                                                                                                                                                     
                                                                                                                                                      - **Context Fields** (when applicable):
                                                                                                                                                      - - user_id (authenticated user)
                                                                                                                                                        - - action (what operation)
                                                                                                                                                          - - resource_type (entity being operated on)
                                                                                                                                                            - - status (success/failure/partial)
                                                                                                                                                             
                                                                                                                                                              - **Error Fields** (for errors):
                                                                                                                                                              - - error_type (exception class)
                                                                                                                                                                - - error_message (error description)
                                                                                                                                                                  - - stack_trace (full traceback)
                                                                                                                                                                    - - error_code (if applicable)

                                                                                                                                                                      ## 4. Tracing Strategy
                                                                                                                                                                      
                                                                                                                                                                      ### 4.1 Trace Sampling
                                                                                                                                                                      
                                                                                                                                                                      **Standard Sampling**:
                                                                                                                                                                      - Normal requests: 10% sampled
                                                                                                                                                                      - - Error requests: 100% sampled
                                                                                                                                                                        - - Slow requests (> 2x p99): 100% sampled
                                                                                                                                                                          - - Admin/test requests: 5% sampled
                                                                                                                                                                           
                                                                                                                                                                            - **Sampling Benefits**:
                                                                                                                                                                            - - Reduces cardinality explosion
                                                                                                                                                                              - - Maintains visibility into issues
                                                                                                                                                                                - - Acceptable resource overhead
                                                                                                                                                                                 
                                                                                                                                                                                  - ### 4.2 Span Instrumentation
                                                                                                                                                                                 
                                                                                                                                                                                  - **Automatic Spans**:
                                                                                                                                                                                  - - HTTP requests (client and server)
                                                                                                                                                                                    - - Database queries
                                                                                                                                                                                      - - Cache operations
                                                                                                                                                                                        - - External API calls
                                                                                                                                                                                          - - Message queue operations
                                                                                                                                                                                           
                                                                                                                                                                                            - **Custom Spans**:
                                                                                                                                                                                            - - Business logic operations
                                                                                                                                                                                              - - Complex calculations
                                                                                                                                                                                                - - Third-party integrations
                                                                                                                                                                                                  - - Data transformations
                                                                                                                                                                                                   
                                                                                                                                                                                                    - **Span Attributes**:
                                                                                                                                                                                                    - - Trace ID (unique request identifier)
                                                                                                                                                                                                      - - Span ID (individual operation)
                                                                                                                                                                                                        - - Parent span ID (call hierarchy)
                                                                                                                                                                                                          - - Operation name (descriptive)
                                                                                                                                                                                                            - - Status (success/failure/error)
                                                                                                                                                                                                              - - Tags (key-value metadata)
                                                                                                                                                                                                               
                                                                                                                                                                                                                - ### 4.3 Trace Correlation
                                                                                                                                                                                                               
                                                                                                                                                                                                                - All requests should include:
                                                                                                                                                                                                                - - `X-Trace-ID`: Unique identifier across all services
                                                                                                                                                                                                                  - - `X-Span-ID`: This span's identifier
                                                                                                                                                                                                                    - - `X-Parent-Span-ID`: Calling span
                                                                                                                                                                                                                      - - `X-Request-ID`: Legacy format for compatibility

                                                                                                                                                                                                                        ## 5. Alerting Strategy
                                                                                                                                                                                                                        
                                                                                                                                                                                                                        ### 5.1 Alert Severity Levels
                                                                                                                                                                                                                        
                                                                                                                                                                                                                        **Critical (P1)**: Page immediately
                                                                                                                                                                                                                        - Service completely unavailable
                                                                                                                                                                                                                        - - Error rate > 10%
                                                                                                                                                                                                                          - - Uptime < 99%
                                                                                                                                                                                                                            - - Data loss occurring
                                                                                                                                                                                                                              - - Security incident in progress
                                                                                                                                                                                                                               
                                                                                                                                                                                                                                - **High (P2)**: Page in 5 minutes
                                                                                                                                                                                                                                - - Major feature broken
                                                                                                                                                                                                                                  - - Error rate 1-10%
                                                                                                                                                                                                                                    - - Latency > 5x baseline
                                                                                                                                                                                                                                      - - Database replication lag > 30 seconds
                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                        - **Medium (P3)**: Alert only
                                                                                                                                                                                                                                        - - Feature partially broken
                                                                                                                                                                                                                                          - - Error rate 0.1-1%
                                                                                                                                                                                                                                            - - Latency 2-5x baseline
                                                                                                                                                                                                                                              - - Resource usage > 80%
                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                - **Low (P4)**: Email/dashboard only
                                                                                                                                                                                                                                                - - Non-critical alerts
                                                                                                                                                                                                                                                  - - Cosmetic issues
                                                                                                                                                                                                                                                    - - Informational messages
                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                      - ### 5.2 Alert Configuration
                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                      - **Alert Rules**:
                                                                                                                                                                                                                                                      - ```
                                                                                                                                                                                                                                                        # Request error rate
                                                                                                                                                                                                                                                        alert: HighErrorRate
                                                                                                                                                                                                                                                        expr: rate(requests_total{status=~"5.."}[5m]) > 0.01
                                                                                                                                                                                                                                                        for: 5m
                                                                                                                                                                                                                                                        severity: P2

                                                                                                                                                                                                                                                        # Latency alert
                                                                                                                                                                                                                                                        alert: HighLatency
                                                                                                                                                                                                                                                        expr: histogram_quantile(0.99, request_duration_seconds) > 0.3
                                                                                                                                                                                                                                                        for: 10m
                                                                                                                                                                                                                                                        severity: P2

                                                                                                                                                                                                                                                        # Resource saturation
                                                                                                                                                                                                                                                        alert: HighCPU
                                                                                                                                                                                                                                                        expr: node_cpu_usage > 0.80
                                                                                                                                                                                                                                                        for: 15m
                                                                                                                                                                                                                                                        severity: P3
                                                                                                                                                                                                                                                        ```
                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                        ### 5.3 Alert Routing
                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                        | Alert | Severity | Channel | Owner |
                                                                                                                                                                                                                                                        |-------|----------|---------|-------|
                                                                                                                                                                                                                                                        | Service down | P1 | PagerDuty | On-call |
                                                                                                                                                                                                                                                        | High error rate | P2 | PagerDuty | On-call |
                                                                                                                                                                                                                                                        | Performance degraded | P3 | Slack | Service lead |
                                                                                                                                                                                                                                                        | Resource alert | P4 | Dashboard | DevOps |
                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                        ### 5.4 Reducing Alert Fatigue
                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                        **Best Practices**:
                                                                                                                                                                                                                                                        - Alert on business impact, not implementation details
                                                                                                                                                                                                                                                        - - Combine related metrics into composite alerts
                                                                                                                                                                                                                                                          - - Use time-based thresholds (at least 5 minutes)
                                                                                                                                                                                                                                                            - - Set appropriate thresholds (avoid hair-trigger sensitivity)
                                                                                                                                                                                                                                                              - - Review alert effectiveness monthly
                                                                                                                                                                                                                                                                - - Silence noisy alerts during known maintenance
                                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                  - **Target**: < 2 pages per engineer per week

                                                                                                                                                                                                                                                                  ## 6. Log Aggregation
                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                  ### 6.1 Log Collection Architecture
                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                  **Flow**: Application  Fluent Bit  Logstash  Elasticsearch
                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                  **Fluent Bit** (lightweight agent):
                                                                                                                                                                                                                                                                  - Runs on every host
                                                                                                                                                                                                                                                                  - - Collects application logs
                                                                                                                                                                                                                                                                    - - Adds metadata (hostname, container_id)
                                                                                                                                                                                                                                                                      - - Forwards to Logstash
                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                        - **Logstash** (log processor):
                                                                                                                                                                                                                                                                        - - Parses log format
                                                                                                                                                                                                                                                                          - - Enriches with additional context
                                                                                                                                                                                                                                                                            - - Filters sensitive data (PII)
                                                                                                                                                                                                                                                                              - - Routes to appropriate index
                                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                - **Elasticsearch**:
                                                                                                                                                                                                                                                                                - - Indexes logs
                                                                                                                                                                                                                                                                                  - - Enables full-text search
                                                                                                                                                                                                                                                                                    - - Aggregates metrics
                                                                                                                                                                                                                                                                                      - - Powers Kibana dashboards
                                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                        - ### 6.2 Index Strategy
                                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                        - **Index Naming**: `logs-{service}-{date}`
                                                                                                                                                                                                                                                                                        - - logs-api-2026-01-30
                                                                                                                                                                                                                                                                                          - - logs-database-2026-01-30
                                                                                                                                                                                                                                                                                            - - logs-payments-2026-01-30
                                                                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                                              - **Retention Policy**:
                                                                                                                                                                                                                                                                                              - - Hot (0-7 days): Full resolution, searchable
                                                                                                                                                                                                                                                                                                - - Warm (7-30 days): Compressed, searchable
                                                                                                                                                                                                                                                                                                  - - Cold (30-90 days): Compressed, slower search
                                                                                                                                                                                                                                                                                                    - - Archive (90+ days): Backup only
                                                                                                                                                                                                                                                                                                      - 
                                                                                                                                                                                                                                                                                                      **Partition Strategy**:
                                                                                                                                                                                                                                                                                                        - By service (for granular access control)
                                                                                                                                                                                                                                                                                                        - - By severity (separate error logs from info)
                                                                                                                                                                                                                                                                                                          - - By environment (prod vs. staging vs. dev)

                                                                                                                                                                                                                                                                                                                                                        - - API health check (GET /)

                                                                                                                                                                                                                                                                                                                                                                    - **Test Location**:
                                                                                                                                                                                                                                                                                                                                                                    - - Multiple geographic regions
                                                                                                                                                                                                                                                                                                                                                                      - - Different network conditions
                                                                                                                                                                                                                                                                                                                                                                        - - Various device types

                                                                                                                                                                                                                                                                                                                                                                                                                      ### 10.1 PII Handling
                                                                                                                                                                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                                                                                                                                      **PII Fields to Redact**:
                                                                                                                                                                                                                                                                                                                                                                                                                      - Email addresses
                                                                                                                                                                                                                                                                                                                                                                                                                      - Phone numbers
                                                                                                                                                                                                                                                                                                                                                                                                                      - Credit card numbers
                                                                                                                                                                                                                                                                                                                                                                                                                      - API keys and tokens
                                                                                                                                                                                                                                                                                                                                                                                                                      - - Social security numbers
                                                                                                                                                                                                                                                                                                                                                                                                                        - - Password hashes
                                                                                                                                                                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                                                                                                                                        **Redaction Strategy**:
                                                                                                                                                                                                                                                                                                                                                                                                                        - Automatic redaction in Logstash
                                                                                                                                                                                                                                                                                                                                                                                                                        - Pattern matching for common PII
                                                                                                                                                                                                                                                                                                                                                                                                                        - - Configuration per field
                                                                                                                                                                                                                                                                                                                                                                                                                        - Audit logging of redactions
                                                                                                                                                                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                                                                                                                                        ### 10.2 Compliance Requirements
                                                                                                                                                                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                                                                                                                                        **GDPR**:
                                                                                                                                                                                                                                                                                                                                                                                                                        - Right to erasure: Delete logs containing specific user
                                                                                                                                                                                                                                                                                                                                                                                                                        - Data retention: Don't retain longer than necessary
                                                                                                                                                                                                                                                                                                                                                                                                                        - - Data portability: Export user's data from logs
                                                                                                                                                                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                                                                                                                                        **SOC 2**:
                                                                                                                                                                                                                                                                                                                                                                                                                        - Log retention: Maintain for audit trail
                                                                                                                                                                                                                                                                                                                                                                                                                        - Access control: Restrict log access
                                                                                                                                                                                                                                                                                                                                                                                                                        - Integrity: Ensure logs cannot be modified
                                                                                                                                                                                                                                                                                                                                                                                                                        - Monitoring: Alert on suspicious access
