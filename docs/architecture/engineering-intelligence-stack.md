# How to Explain This Diagram (30 seconds)

When presenting it, the explanation should be extremely simple:

> “Summit observes engineering systems, builds a structural model of the repository, predicts how the system will evolve, and then helps guide engineering decisions.”

This explanation maps directly to the layers.

---

# Layer Explanation

## Engineering Signals

Sources of truth:

* code repositories
* pull requests
* CI pipelines
* dependency graphs
* agent activity

This layer answers:

**“What is happening in the engineering system?”**

---

## Repository Knowledge Graph

Summit converts signals into **structure**.

It models:

* subsystems
* dependencies
* CI behavior
* change patterns

This layer answers:

**“How is the repository structured?”**

---

## Engineering Intelligence Layer

This is where Summit becomes powerful.

Capabilities include:

* CI failure prediction
* architecture simulation
* dependency cascade modeling
* subsystem health analysis

This layer answers:

**“What is likely to happen next?”**

---

## Autonomous Engineering Strategy

Using the Digital Twin and Intelligence Graph, Summit determines:

* which subsystems are fragile
* which refactors yield the most benefit
* where engineering effort should focus

This layer answers:

**“What should we do?”**

---

## Autonomous Engineering Governance

The final layer converts strategy into **actionable artifacts**:

* PR proposals
* architecture change plans
* roadmap updates

This layer answers:

**“How should we execute the strategy?”**

Engineers immediately recognize this pattern.
