# E-Commerce UX Evaluation Data Handling

## Scope

This policy governs telemetry and evidence generated by Summit e-commerce UX evaluation modules.

## Data Classification

- **Allowed:** Aggregate UX metrics, threshold decisions, deterministic hashes, gate outcomes.
- **Prohibited:** PII, payment data, session identifiers, auth tokens, free-form user input payloads.

## Logging Requirements

- Log aggregate metrics only.
- Redact identifier-bearing fields before persistence.
- Preserve deterministic artifact structure (`report.json`, `metrics.json`, `stamp.json`).

## Retention

- Default retention window: 30 days for aggregate UX evaluation outputs.
- Extended retention requires governance approval and documented rationale.

## Enforcement

- Deny-by-default for unknown fields in UX evidence payloads.
- CI must fail if prohibited data classes are detected in evaluation artifacts.
