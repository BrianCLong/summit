# InfoWar Analytics Guardrails

## 1. No-Offensive Use Statement
Summit's INFOWAR module is strictly for defensive OSINT, governance, and incident response. It must NOT be used to enable influence operations, microtargeting, or evasion of detection.

## 2. Uncertainty Policy
All claims MUST include a confidence score (0.0 to 1.0) and be linked to at least one Evidence ID.

## 3. Redaction Policy
PII, private handles, and doxxing strings MUST be redacted before ingestion into narrative graphs.

## 4. Audit Export Format
Audit trails must be exported in the canonical INFOWAR_AUDIT_V1 format, including hash-chained receipts.
