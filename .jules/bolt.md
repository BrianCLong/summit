## 2024-06-05 - [Batch Database Insertions for Comments]
**Learning:** Refactoring individual loop-based database insertions into multi-row batch inserts with chunking significantly reduces round-trip latency (from O(N) to O(1) per chunk). In this codebase, the `pg` library supports this pattern using manual placeholder generation. Standalone scripts using `tsx` are more reliable for verifying these optimizations when the main test suite is prone to timeouts.
**Action:** Always look for loops containing `await client.query('INSERT ...')` and refactor them to use the batch insertion pattern with a chunk size of 100. Verify using standalone query-counting scripts if Jest fails.
