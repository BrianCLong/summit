# SLO Dashboards

## SLO Definitions (Tri-Pane Flow)
- **Availability:** 99.5% monthly for preview + execution endpoints (error budget: 216 minutes).
- **Latency:** P50 400 ms / P95 850 ms for preview; P50 700 ms / P95 1.5 s for execution.
- **Accuracy:** ≥95% precision on curated NL→Cypher fixtures; error budget resets weekly.
- **$/Insight:** Target <$2.50 per insight for demo tenants with seeded data.

## Dashboard Layout (Grafana)
1. **Overview:** uptime gauge, error budget burn chart, executive summary text panel.
2. **Preview Performance:** latency percentiles, guardrail block counts, cache hit rate.
3. **Ledger Health:** ingest success %, hash chain verification status, export duration histogram.
4. **FinOps:** $/insight, quota consumption, cost guard block/allow timeline with annotations.
5. **Reliability:** chaos drill markers, R1/R3 status notes, XAI overlay for `Explain this view` feedback.

## JSON Hook
- Export the dashboard JSON via Grafana API `GET /api/dashboards/uid/{uid}`.
- Store exports under `ops/dashboards/{date}-tri-pane.json` with checksum; include link in exec decks.
- Parameters to template: `tenant`, `environment`, `service` for reuse across sandboxes.

## Alerting
- Alert when preview P95 > 850 ms for 3 consecutive windows.
- Alert when ledger verification failures > 0 in a 6-hour window.
- Alert when $/insight > $2.50 for demo tenant or cost guard blocks > 5 per hour (possible false positives).

## Runbook Links
- R1 triage for preview latency regressions.
- R2 verification export steps for demos.
- R3 rollback for ledger or cost guard outages.

## Data Sources
- OpenTelemetry metrics scraped by Prometheus -> Grafana.
- Billing metrics via FinOps exporter (USD normalized) with cost guard annotations.
- Demo data generated by `scripts/generate_demo_data.py` to seed synthetic usage.
