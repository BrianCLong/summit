# Claim Registry - Maven: Claude Code for PMs

## MAVEN-CLAUDE-PM Claims

- MAVEN-CLAUDE-PM-001: Prompt engineering for analytical workflows allows PMs to transform ambiguous questions into structured, reproducible queries.
- MAVEN-CLAUDE-PM-002: Claude can assist in working with diverse datasets (CSV, SQL exports, analytics dumps) for rapid exploratory analysis.
- MAVEN-CLAUDE-PM-003: AI-assisted scripting and automation reduces dependency on engineering for lightweight internal tooling.
- MAVEN-CLAUDE-PM-004: Reproducible decision artifacts backed by data increase decision velocity and transparency.
- MAVEN-CLAUDE-PM-005: Lightweight dashboards can be built and deployed within a single sprint using AI assistance.
- MAVEN-CLAUDE-PM-006: Experiment design and documentation can be automated via structured prompts and LLM-assisted drafting.

## SUMMIT_ORIGINAL Subsumption Claims

- SUMMIT-PM-TRANS-001: Analytical prompts must be versioned and governed in the `prompts/analytical/` registry.
- SUMMIT-PM-TRANS-002: All data-to-decision workflows must produce a signed `EvidenceBundle` for auditability.
- SUMMIT-PM-TRANS-003: Lightweight PM dashboards must be deployed as sandboxed components in the `apps/pm-dashboards/` subspace.
- SUMMIT-PM-TRANS-004: Decision memos must utilize the `DecisionLedger` service for immutability.
