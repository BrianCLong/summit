# Summit Transcendence Plan: Claude Code for PMs (Data → Decisions)

## 1. Transcendence Work: Beyond "Claude Code for PMs"

To surpass the original Maven course material, the Summit platform will implement the following net-new capabilities:

### Transcendence A: Graph-Native PM Analysis
- **Innovation:** Integrate the Maven "Data → Decisions" patterns into the **IntelGraph** Cypher logic.
- **Wedge:** Instead of simple CSV/SQL, PMs can query the **Knowledge Graph** directly using `nl2cypher` with pre-defined "PM-Intents".
- **Gating Plan:** `verify-graph-intent-pm-budget` - ensures PM queries do not exceed graph traversal limits.

### Transcendence B: Deterministic Decision Provenance
- **Innovation:** Link every Claude-generated insight to its exact **Evidence ID** in the `provenance-ledger`.
- **Wedge:** While the Maven course focuses on manual reproducibility, Summit automates the **signing and attestation** of data-backed insights.
- **Gating Plan:** `prov-ledger-insight-attestation` - insight PRs fail without a valid `EvidenceBundle`.

### Transcendence C: Multi-Agent Consensus for PM Decisions
- **Innovation:** Run multiple analytical models (e.g., Claude, GPT-4, Llama-3) on the same dataset to identify **disagreements or hallucinations** in PM reports.
- **Wedge:** The course assumes a single assistant. Summit provides a **consensus layer** to identify analytical drift.
- **Gating Plan:** `cross-model-consensus-gate` - reports fail if models disagree significantly on key metrics (e.g., churn rate, conversion).

### Transcendence D: Policy-Native Dashboard Controls
- **Innovation:** Dashboard components generated by PMs are automatically wrapped in **OPA policies**.
- **Wedge:** Move from "lightweight dashboards" to **Governed Dashboards** that enforce data residency and RBAC at the component level.
- **Gating Plan:** `opa-ui-component-verify` - prevents unauthorized data exposure in PM-created views.

## 2. Moat & Gate Strategy

By converting these innovations into hard control points (CI gates), Summit creates a moat that competitor PM tools (which lack a native governance layer) cannot easily cross.

| Feature | Moat Type | Gate Locator |
| --- | --- | --- |
| Insight Attestation | Evidence Moat | `scripts/ci/verify_insight_attestation.ts` |
| Graph-Intent PM | Architectural Moat | `packages/intent-compiler/pm-rules.rego` |
| Model Consensus | Security Moat | `services/consensus-service/validator.py` |
| Policy-Native UI | Governance Moat | `packages/governance/policy/ui-gates.rego` |
