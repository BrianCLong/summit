# Summit Architecture Map

**Date**: 2026-03-05
**Reference**: Luma Creative Agents / Unified Intelligence Subsumption

## 1. Executive Summary
Summit is evolving from an agentic OSINT platform into a unified governed orchestration + simulation platform. This architecture map establishes the blueprint for subsuming multimodal intelligence and synthetic media/simulation workloads inside `BrianCLong/summit`.

## 2. Layered Architecture Stack
The target state of the repo consolidates into these execution layers:

### API + Experience Layer
Existing entry points (`api/`, SDK, UX surfaces) remain the front door.

### Agentic Control Plane
The convergence routing layer. Workloads are directed based on capability requirements to:
- **Creative Agents**: simple multimodal tool orchestration.
- **Creative OS**: complex production DAGs and specialist agent assembly.
- **Cognitive Media Engine (CME)**: scene simulation and world-model-based media generation.
- **Synthetic Reality Engine (SRE)**: general purpose synthetic environments and agent ecosystems.

### Memory + Retrieval
**GraphRAG** remains central, holding both semantic context for traditional OSINT and provenance/world-state for the SRE and CME layers.

### Governance + CI
Every layer strictly subordinates to verifiable determinism. All artifact outputs must be hash-stable, signed, and gate-checked in CI.

## 5. Convergence Protocol
All new capability modules (`creative-os`, `cognitive-media`, `sre`) must:
1. Define clear API contracts.
2. Produce deterministic artifacts (e.g. `report.json`, `metrics.json`, `stamp.json`).
3. Contain their own threat models in `tests/security/`.

## 6. Required Evidence IDs
All artifacts generated by these layers must bear `Evidence IDs` following:
`EVID-<MODULE>-<HASH>` (e.g., `EVID-SRE-a1b2c3d4`)
