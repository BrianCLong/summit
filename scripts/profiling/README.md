# Summit Profiling Tools

This directory contains standalone diagnostic scripts for profiling CPU and memory usage of the Summit Python backend. These scripts are designed to help identify performance bottlenecks, memory leaks, and high memory usage, without requiring modifications to production source code.

## Tools

* **`profile_memory.py`**: Uses Python's built-in `tracemalloc` module to trace memory allocations, generate a snapshot of memory usage, and optionally produce a heap dump.
* **`profile_cpu.py`**: Uses Python's built-in `cProfile` module to measure execution time of functions, identifying CPU-bound bottlenecks and producing statistics readable by profiling visualizers.
* **`generate_flamegraph.py`**: A helper script that processes `.prof` files output by `profile_cpu.py` to generate an HTML visualization of the CPU profile and prepare data for flame graph generation.

## Usage Guide

All output artifacts are saved by default to the `artifacts/profiling/` directory.

### 1. Memory Profiling (`profile_memory.py`)

Memory profiling traces where memory is being allocated in your code. It's particularly useful for diagnosing memory leaks in long-running processes or high memory usage during the GraphRAG pipeline execution.

**Profile a standalone script:**
```bash
./scripts/profiling/profile_memory.py --script scripts/some_script.py [args...]
```

**Profile a Python module:**
```bash
./scripts/profiling/profile_memory.py --module summit.graph.operations [args...]
```

**Outputs:**
* `memory_profile_*.txt`: A text file containing the top 20 memory allocations, comparing the memory state before and after execution.
* `heap_dump_*.json`: A JSON summary of the top 100 memory allocations (file, line number, size, count), useful for external analysis.

### 2. CPU Profiling (`profile_cpu.py`)

CPU profiling measures the time spent in each function. It is essential for finding performance bottlenecks (e.g., slow data processing, inefficient database queries).

**Profile a standalone script:**
```bash
./scripts/profiling/profile_cpu.py --script scripts/some_script.py [args...]
```

**Profile a Python module:**
```bash
./scripts/profiling/profile_cpu.py --module summit.graph.operations [args...]
```

**Outputs:**
* `cpu_profile_*.txt`: A text summary of the top 20 functions by cumulative execution time.
* `cpu_profile_*.prof`: The raw `cProfile` statistics file. This can be viewed with visualizers like `snakeviz`.
* `call_graph_*.json`: A JSON file containing simplified call graph data (function names, call counts, total time, cumulative time) for the top 100 functions.

### 3. Flame Graph Generation (`generate_flamegraph.py`)

This script takes the `.prof` file generated by `profile_cpu.py` and creates an HTML report and a folded stack file that can be used to generate flame graphs.

```bash
./scripts/profiling/generate_flamegraph.py artifacts/profiling/cpu_profile_*.prof
```

**Outputs:**
* `*.html`: An interactive HTML report visualizing the top functions and their execution time percentage.
* `*.folded`: A folded stack format file. While this script provides a basic implementation, for true flame graphs it is recommended to use specialized tools like `py-spy`.

## Common Scenarios

### Profiling the GraphRAG Pipeline

To profile the GraphRAG pipeline, you can run the evaluation harness or a specific test script using the CPU profiler:

```bash
# CPU Profiling
./scripts/profiling/profile_cpu.py --script evals/retrieval/eval_harness.py

# Generate visualization
./scripts/profiling/generate_flamegraph.py artifacts/profiling/cpu_profile_eval_harness_*.prof
```

### Detecting Memory Leaks in Batch Jobs

If a batch job is consuming too much memory, use the memory profiler to identify the source:

```bash
./scripts/profiling/profile_memory.py --module summit.jobs.batch_processor
```

Review the resulting `memory_profile_*.txt` to see which files and lines of code are responsible for the largest memory allocations.

## Advanced Visualization (External Tools)

While these scripts provide built-in reports, you can use standard Python tools for deeper analysis:

* **SnakeViz**: A browser-based graphical viewer for the output of Python's cProfile module.
  ```bash
  pip install snakeviz
  snakeviz artifacts/profiling/cpu_profile_*.prof
  ```

* **Py-Spy**: A sampling profiler for Python programs that can generate flame graphs directly.
  ```bash
  # Example usage (requires sudo on some systems)
  py-spy record -o profile.svg -- python your_script.py
  ```
