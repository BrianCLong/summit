# Security Policy

## Reporting a Vulnerability

The Summit team takes security seriously. If you discover a security vulnerability in this project, please report it to us as soon as possible.

### Contact Information
Please send security reports to **security@summit.ai**.

### Disclosure Process
1. **Initial Response:** We will acknowledge receipt of your report within 24 hours.
2. **Triage:** We will investigate and validate the vulnerability. We aim to provide a timeline for remediation within 72 hours.
3. **Remediation:** We will work on a fix as a high priority.
4. **Public Disclosure:** We request that you do not disclose the vulnerability publicly until we have had a chance to fix it (typically within 30 days).

## Supported Versions
Only the latest major version of the Summit platform is actively supported for security updates.

## Guidelines
- Avoid performing any actions that could disrupt the service for other users.
- Do not attempt to access or modify data that does not belong to you.
- Provide detailed steps to reproduce the issue in your report.

## Safe Harbor
We will not take legal action against researchers who discover and report vulnerabilities in good faith and in accordance with this policy.
